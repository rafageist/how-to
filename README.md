# How-To Knowledge Base

Public repository for practical Markdown documentation extracted from solved technical cases.

This repository does not store raw chat conversations. It stores reusable, cleaned technical knowledge: context, problem, solution, commands, scripts, troubleshooting notes, and operational rules.

## Purpose

The goal is to preserve useful knowledge from problem-solving sessions so it can be reused later.

Expected flow:

```text
real case / conversation
        ↓
clean technical summary
        ↓
reproducible Markdown how-to
        ↓
versioned GitHub documentation
